# Prompt 书写规范

本文档定义了项目中所有 Prompt 的书写规范，确保 Prompt 的一致性、可复用性和可维护性。

## 1. 文件命名规范

```
[序号]_[类型]_[功能描述].md
```

示例：
- `01_req_需求拆解.md`
- `02_design_架构设计.md`
- `03_impl_API接口生成.md`

## 2. Prompt 模板结构

每个 Prompt 文件应包含以下部分：

```markdown
# [Prompt 名称]

## 元信息
- **版本**: v1.0.0
- **作者**: [作者名]
- **创建日期**: YYYY-MM-DD
- **最后更新**: YYYY-MM-DD
- **适用模型**: GPT-4 / Claude / 通用

## 目的
[简要描述此 Prompt 的用途]

## 输入变量
| 变量名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| {{VAR_NAME}} | string | 是 | 变量说明 |

## Prompt 正文
[实际的 Prompt 内容]

## 输出格式约束
[期望的输出格式说明]

## 使用示例
[输入输出示例]

## 注意事项
[使用时的注意点]
```

## 3. 变量命名规范

- 使用双大括号包裹变量：`{{VARIABLE_NAME}}`
- 变量名使用大写字母和下划线
- 变量名应具有描述性

常用变量：
- `{{CONTEXT}}` - 上下文信息
- `{{REQUIREMENTS}}` - 需求描述
- `{{CODE_SNIPPET}}` - 代码片段
- `{{ERROR_MESSAGE}}` - 错误信息
- `{{LANGUAGE}}` - 编程语言

## 4. 输出格式约束

### 4.1 结构化输出

推荐使用 JSON 或 Markdown 格式约束输出：

```
请以 JSON 格式输出，包含以下字段：
{
  "summary": "简要总结",
  "details": ["详细点1", "详细点2"],
  "confidence": 0.0-1.0
}
```

### 4.2 分步输出

对于复杂任务，要求分步输出：

```
请按以下步骤输出：
1. 分析：...
2. 方案：...
3. 实现：...
4. 验证：...
```

## 5. 版本管理

- 每次重大修改需更新版本号
- 过期的 Prompt 移至 `99_archive/` 目录
- 在 Prompt 文件中记录变更历史

## 6. 质量检查清单

在提交 Prompt 前，请确认：

- [ ] 目的明确
- [ ] 变量定义完整
- [ ] 输出格式有约束
- [ ] 包含使用示例
- [ ] 经过实际测试
- [ ] 版本信息完整
