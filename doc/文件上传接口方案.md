# Spring Boot 通用文件上传接口 — 完整实现教程

## 一、背景

项目需要一个通用的文件上传接口，上传成功后返回文件的访问路径，供前端或其他模块使用。

## 二、目标

- 支持任意类型文件上传（图片、文档、PDF 等）
- 上传成功返回文件访问路径
- 限制单文件大小，防止滥用
- 配置可灵活切换（本地存储 / 对象存储）

## 三、技术选型

| 项目 | 方案 | 说明 |
|------|------|------|
| 文件接收 | `MultipartFile` | Spring Boot 原生支持 |
| 存储方式 | 本地磁盘（初始） | 后期可扩展 OSS / MinIO |
| 工具库 | Hutool `FileUtil` | 已引入，避免重复造轮子 |
| 文件名生成 | UUID | 防止中文名 / 重名冲突 |

## 四、接口设计

### 4.1 请求

```
POST /api/files/upload
Content-Type: multipart/form-data
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `file` | MultipartFile | 是 | 上传的文件 |

### 4.2 响应

**成功：**

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "url": "/files/2026/07/02/a1b2c3d4-xxx.jpg",
    "originalName": "photo.jpg",
    "size": 102400
  }
}
```

**失败：**

```json
{
  "code": 400,
  "message": "文件不能为空",
  "data": null
}
```

### 4.3 约束

| 约束项 | 默认值 | 配置键 |
|--------|--------|--------|
| 单文件最大 | 10 MB | `app.file.max-size` |
| 允许扩展名 | jpg, png, gif, pdf, doc, docx, xls, xlsx, txt, zip | `app.file.allowed-extensions` |
| 存储根目录 | `./uploads` | `app.file.upload-dir` |

### 5.2 第一步：配置类 — `FileProperties.java`

**路径：** `src/main/java/com/example/bookmanage/common/config/FileProperties.java`

**作用：** 读取 `application.yml` 中 `app.file.*` 的配置，集中管理上传参数，避免硬编码。

```java
package com.example.bookmanage.common.config;

import lombok.Data;
import org.springframework.boot.context.properties.ConfigurationProperties;
import org.springframework.stereotype.Component;

import java.util.Arrays;
import java.util.List;

/**
 * 文件上传配置
 *
 * 对应 application.yml 中 app.file.* 的配置项
 */
@Data
@Component
@ConfigurationProperties(prefix = "app.file")
public class FileProperties {

    /**
     * 上传文件存储根目录
     * 默认值: ./uploads（项目运行目录下的 uploads 文件夹）
     * 生产环境建议改为绝对路径，如 /data/uploads
     */
    private String uploadDir = "./uploads";

    /**
     * 单文件最大大小（字节）
     * 默认值: 10MB
     */
    private long maxSize = 10 * 1024 * 1024;

    /**
     * 允许上传的文件扩展名白名单
     * 小写匹配，不带点号
     */
    private List<String> allowedExtensions = Arrays.asList(
            "jpg", "png", "gif", "bmp", "webp",
            "pdf",
            "doc", "docx", "xls", "xlsx", "ppt", "pptx",
            "txt", "csv",
            "zip", "rar", "7z"
    );
}
```

**知识点说明：**

- `@ConfigurationProperties(prefix = "app.file")`：将 yml 中 `app.file` 下的属性自动绑定到该类的字段
- `@Component`：注册为 Spring Bean，可在其他类中注入
- 默认值：代码中写的是代码默认值，yml 中的值会覆盖它

### 5.5 第四步：Service 实现 — `FileServiceImpl.java`

**路径：** `src/main/java/com/example/bookmanage/service/impl/FileServiceImpl.java`

**作用：** 核心业务逻辑 — 校验文件、生成路径、写入磁盘、返回结果。

```java
package com.example.bookmanage.service.impl;

import cn.hutool.core.io.FileUtil;
import cn.hutool.core.util.IdUtil;
import com.example.bookmanage.common.config.FileProperties;
import com.example.bookmanage.common.exception.BusinessException;
import com.example.bookmanage.common.exception.ErrorCode;
import com.example.bookmanage.model.vo.FileUploadVO;
import com.example.bookmanage.service.FileService;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.stereotype.Service;
import org.springframework.web.multipart.MultipartFile;

import java.io.File;
import java.io.IOException;
import java.time.LocalDate;
import java.time.format.DateTimeFormatter;

/**
 * 文件服务实现 — 本地磁盘存储
 */
@Slf4j
@Service
@RequiredArgsConstructor
public class FileServiceImpl implements FileService {

    private static final DateTimeFormatter DATE_DIR_FORMAT = DateTimeFormatter.ofPattern("yyyy/MM/dd");

    private final FileProperties fileProperties;

    @Override
    public FileUploadVO upload(MultipartFile file) {
        // 1. 校验文件非空
        validateFileNotEmpty(file);

        // 2. 校验文件大小
        validateFileSize(file);

        // 3. 校验扩展名白名单
        String extension = getAndValidateExtension(file);

        // 4. 生成存储路径
        String relativePath = generateRelativePath(extension);

        // 5. 写入磁盘
        saveToFileSystem(file, relativePath);

        // 6. 构造返回结果
        String accessUrl = "/files/" + relativePath;
        log.info("文件上传成功: originalName={}, url={}, size={}",
                file.getOriginalFilename(), accessUrl, file.getSize());

        return new FileUploadVO()
                .setUrl(accessUrl)
                .setOriginalName(file.getOriginalFilename())
                .setSize(file.getSize());
    }

    /**
     * 校验文件非空
     */
    private void validateFileNotEmpty(MultipartFile file) {
        if (file == null || file.isEmpty()) {
            throw new BusinessException(ErrorCode.PARAM_ERROR, "文件不能为空");
        }
    }

    /**
     * 校验文件大小
     */
    private void validateFileSize(MultipartFile file) {
        if (file.getSize() > fileProperties.getMaxSize()) {
            long maxSizeMB = fileProperties.getMaxSize() / (1024 * 1024);
            throw new BusinessException(ErrorCode.PARAM_ERROR,
                    "文件大小超过限制，最大允许 " + maxSizeMB + " MB");
        }
    }

    /**
     * 获取并校验文件扩展名
     *
     * @return 小写的扩展名（不含点号）
     */
    private String getAndValidateExtension(MultipartFile file) {
        String originalName = file.getOriginalFilename();
        if (originalName == null || !originalName.contains(".")) {
            throw new BusinessException(ErrorCode.PARAM_ERROR, "文件缺少扩展名");
        }

        // 截取最后一个点号之后的内容，转小写
        String extension = originalName.substring(originalName.lastIndexOf('.') + 1).toLowerCase();

        if (!fileProperties.getAllowedExtensions().contains(extension)) {
            throw new BusinessException(ErrorCode.PARAM_ERROR,
                    "不支持的文件类型: ." + extension + "，允许: " + fileProperties.getAllowedExtensions());
        }

        return extension;
    }

    /**
     * 生成相对存储路径
     * 格式: yyyy/MM/dd/{uuid}.{ext}
     *
     * @param extension 文件扩展名（不含点号）
     * @return 相对路径
     */
    private String generateRelativePath(String extension) {
        // 按日期创建子目录，避免单目录文件过多
        String dateDir = LocalDate.now().format(DATE_DIR_FORMAT);
        // 使用 SimpleUUID 生成短 UUID，避免中文和特殊字符
        String uuid = IdUtil.simpleUUID();
        return dateDir + "/" + uuid + "." + extension;
    }

    /**
     * 将文件写入磁盘
     *
     * @param file         上传的文件
     * @param relativePath 相对路径
     */
    private void saveToFileSystem(MultipartFile file, String relativePath) {
        // 拼接完整路径
        String fullPath = fileProperties.getUploadDir() + "/" + relativePath;

        // Hutool 工具：自动创建父目录
        FileUtil.mkParentDirs(fullPath);

        try {
            // 写入文件
            file.transferTo(new File(fullPath));
        } catch (IOException e) {
            log.error("文件写入失败: path={}", fullPath, e);
            throw new BusinessException(ErrorCode.SYSTEM_ERROR, "文件保存失败，请重试");
        }
    }
}
```

**知识点说明：**

- `@RequiredArgsConstructor`：Lombok 注解，自动为 `final` 字段生成构造函数，替代 `@Autowired`
- `IdUtil.simpleUUID()`：Hutool 提供，生成 32 位无横线 UUID，比 `UUID.randomUUID()` 更简洁
- `FileUtil.mkParentDirs()`：Hutool 工具，自动创建父目录，等价于 `file.getParentFile().mkdirs()`
- `file.transferTo()`：Spring 的 `MultipartFile` 内置方法，将内存中的文件写入磁盘
- 日期子目录（`yyyy/MM/dd`）：分散存储，避免单个目录下文件数过多导致性能问题

### 5.6 第五步：Controller — `FileController.java`

**路径：** `src/main/java/com/example/bookmanage/controller/FileController.java`

**作用：** 接收 HTTP 请求，调用 Service，返回统一格式响应。

```java
package com.example.bookmanage.controller;

import com.example.bookmanage.common.response.ApiResponse;
import com.example.bookmanage.model.vo.FileUploadVO;
import com.example.bookmanage.service.FileService;
import lombok.RequiredArgsConstructor;
import org.springframework.web.bind.annotation.PostMapping;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RequestParam;
import org.springframework.web.bind.annotation.RestController;
import org.springframework.web.multipart.MultipartFile;

/**
 * 文件上传接口
 */
@RestController
@RequestMapping("/api/files")
@RequiredArgsConstructor
public class FileController {

    private final FileService fileService;

    /**
     * 上传单个文件
     *
     * @param file 上传的文件（form-data 中 key 为 "file"）
     * @return 上传结果，包含文件访问路径
     */
    @PostMapping("/upload")
    public ApiResponse<FileUploadVO> upload(@RequestParam("file") MultipartFile file) {
        FileUploadVO result = fileService.upload(file);
        return ApiResponse.success(result);
    }
}
```

**知识点说明：**

- `@RequestParam("file")`：从 multipart/form-data 中获取名为 `file` 的文件字段
- `ApiResponse.success(result)`：复用项目已有的统一响应格式

### 5.8 第七步：修改 `WebMvcConfig.java` — 静态资源映射

**路径：** `src/main/java/com/example/bookmanage/common/config/WebMvcConfig.java`

**作用：** 将 HTTP 路径 `/files/**` 映射到磁盘上的 `uploadDir` 目录，使上传的文件可通过浏览器直接访问。

```java
package com.example.bookmanage.common.config;

import com.example.bookmanage.common.auth.AuthInterceptor;
import lombok.RequiredArgsConstructor;
import org.springframework.context.annotation.Configuration;
import org.springframework.web.servlet.config.annotation.InterceptorRegistry;
import org.springframework.web.servlet.config.annotation.ResourceHandlerRegistry;
import org.springframework.web.servlet.config.annotation.WebMvcConfigurer;

/**
 * Web MVC 配置
 */
@Configuration
@RequiredArgsConstructor
public class WebMvcConfig implements WebMvcConfigurer {

    private final AuthInterceptor authInterceptor;
    private final FileProperties fileProperties;

    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        registry.addInterceptor(authInterceptor)
                .addPathPatterns("/api/**")
                .excludePathPatterns("/api/auth/login");
    }

    /**
     * 静态资源映射：/files/** -> 磁盘 uploadDir
     *
     * 上传后的文件可通过 http://localhost:8080/files/yyyy/MM/dd/xxx.jpg 直接访问
     * 注意路径末尾的 "/" 不能省略
     */
    @Override
    public void addResourceHandlers(ResourceHandlerRegistry registry) {
        String location = "file:" + fileProperties.getUploadDir() + "/";
        registry.addResourceHandler("/files/**")
                .addResourceLocations(location);
    }
}
```

**与原文件的变更点：**

- 新增注入 `FileProperties`
- 新增 `addResourceHandlers` 方法
- 原有 `addInterceptors` 方法保持不变

```java
                    "file", name, "application/octet-stream", "data".getBytes()

            FileUploadVO result = fileService.upload(file);
            assertNotNull(result.getUrl());
        }
    }
}
```

**测试知识点说明：**

- `@TempDir`：JUnit 5 内置，自动创建临时目录并在测试结束后清理，不污染项目目录
- `MockMultipartFile`：Spring Test 提供，模拟 `MultipartFile`，无需真实 HTTP 请求
- `@Mock` + `@InjectMocks`：Mockito 注解，`@Mock` 创建假的 `FileProperties`，`@InjectMocks` 将其注入 `FileServiceImpl`
- `@DisplayName`：中文描述测试意图，提高可读性

## 九、常见问题

### Q1: 出现 `FileSizeLimitExceededException`

**原因：** Spring 的 `spring.servlet.multipart.max-file-size` 小于 `app.file.max-size`。

**解决：** 确保 yml 中 `max-file-size >= app.file.max-size`。

### Q2: 上传中文文件名乱码

**解决：** 存储时使用 UUID 作为文件名（已实现），原始文件名仅作为返回字段，不参与存储。

### Q3: 生产环境上传目录在哪？

**解决：** 通过环境变量 `FILE_UPLOAD_DIR` 指定绝对路径，如：

```bash
export FILE_UPLOAD_DIR=/data/uploads
```

### Q4: 如何支持多文件上传？

在 Controller 中扩展：

```java
@PostMapping("/batch-upload")
public ApiResponse<List<FileUploadVO>> batchUpload(
        @RequestParam("files") List<MultipartFile> files) {
    List<FileUploadVO> results = files.stream()
            .map(fileService::upload)
            .collect(Collectors.toList());
    return ApiResponse.success(results);
}
```
