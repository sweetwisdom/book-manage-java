这些都是 Maven 依赖，但作用层级不一样。

---

## 1. spring-boot-starter

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter</artifactId>
</dependency>
```

这是 **Spring Boot 基础启动器**。

包含：

* Spring Core
* Spring Context
* Spring Beans
* Spring Boot 自动配置
* 日志框架（Logback + SLF4J）

相当于：

```java
@Component
@Service
@Configuration
@Autowired
```

这些最基础的 Spring 功能能用。

但：

❌ 没有 Web

❌ 没有 MVC

❌ 没有 Tomcat

❌ 没有数据库

适合：

```text
定时任务
命令行程序
后台服务
```

---

## 2. spring-boot-starter-validation

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-validation</artifactId>
</dependency>
```

提供：

```java
@NotNull
@NotBlank
@Size
@Email
@Pattern
```

等参数校验能力。

例如：

```java
@Data
public class UserDTO {

    @NotBlank
    private String name;

    @Min(18)
    private Integer age;
}
```

Controller：

```java
@PostMapping
public void save(@Valid @RequestBody UserDTO dto) {
}
```

请求进来自动校验。

底层：

```text
Hibernate Validator
Jakarta Validation
```

## 3. spring-boot-starter-test

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-test</artifactId>
    <scope>test</scope>
</dependency>
```

测试专用。

包含：

* JUnit 5
* Mockito
* AssertJ
* Spring Test

例如：

```java
@SpringBootTest
class UserServiceTest {

    @Autowired
    UserService userService;

    @Test
    void test() {
    }
}
```

---

### 为什么有

```xml
<scope>test</scope>
```

表示：

```text
编译时不用
运行时不用
测试时才加载
```

打包：

```bash
mvn package
```

不会进入最终 jar。

---

## 4. spring-boot-starter-web

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
</dependency>
```

Web 项目核心依赖。

自动带：

* Spring MVC
* Jackson
* Embedded Tomcat
* JSON转换器

有了它才能写：

```java
@RestController
@RequestMapping("/user")
```

例如：

```java
@GetMapping("/hello")
public String hello() {
    return "hello";
}
```

启动后：

```text
http://localhost:8080/hello
```

即可访问。

---

### 它内部依赖

大概等于：

```text
spring-web
spring-webmvc
jackson
tomcat
```

所以很多项目只写：

```xml
spring-boot-starter-web
```

就够了。

## 5. mybatis-plus-boot-starter

```xml
<dependency>
    <groupId>com.baomidou</groupId>
    <artifactId>mybatis-plus-boot-starter</artifactId>
    <version>3.5.8</version>
</dependency>
```

这是 MyBatis Plus 启动器。

自动配置：

* SqlSessionFactory
* Mapper扫描
* MyBatis配置

让你直接写：

```java
@Mapper
public interface UserMapper
        extends BaseMapper<User> {
}
```

Service：

```java
@Service
public class UserService
        extends ServiceImpl<UserMapper, User> {
}
```

甚至：

```java
userService.list();
userService.save(user);
```

都能直接用。

## 依赖关系图

```text
spring-boot-starter
│
├── 基础Spring能力
│
├── spring-boot-starter-web
│     ├── MVC
│     ├── Jackson
│     └── Tomcat
│
├── spring-boot-starter-validation
│     └── 参数校验
│
├── spring-boot-starter-test
│     └── 测试框架
│
└── mybatis-plus-boot-starter
      ├── MyBatis
      ├── Mapper
      └── CRUD增强
```

---

对于一个典型的 Spring Boot + MyBatis-Plus 后端项目，通常最重要的是：

```xml
spring-boot-starter-web
spring-boot-starter-validation
mybatis-plus-boot-starter
mysql-connector-j
```

而 `spring-boot-starter` 往往可以省略，因为 `spring-boot-starter-web` 已经间接依赖了它。很多项目保留它只是为了表达“这是一个 Spring Boot 应用”。
