## 方案1：SpringBootTest（推荐新手）

测试真实数据库操作。

### UserServiceTest.java

```java
package com.example.bookmanage.service;

import com.example.bookmanage.model.dto.UserDTO;
import com.example.bookmanage.model.vo.UserVO;
import org.junit.jupiter.api.Test;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.boot.test.context.SpringBootTest;

import java.util.List;

@SpringBootTest
class UserServiceTest {

    @Autowired
    private UserService userService;

    @Test
    void getAllUsers() {

        List<UserVO> users = userService.getAllUsers();

        System.out.println(users);

        assert users != null;
    }

    @Test
    void getUserById() {

        UserVO user = userService.getUserById(1L);

        System.out.println(user);

        assert user != null;
    }

    @Test
    void createUser() {

        UserDTO dto = new UserDTO();

        dto.setName("Tom");
        dto.setAge(18);

        UserVO user = userService.createUser(dto);

        System.out.println(user);

        assert user != null;
        assert user.getId() != null;
    }
}
```

## 推荐改成 AssertJ

Spring Boot 默认带：

```java
import static org.assertj.core.api.Assertions.assertThat;
```

例如：

```java
@Test
void createUser() {

    UserDTO dto = new UserDTO();

    dto.setName("Tom");
    dto.setAge(18);

    UserVO user = userService.createUser(dto);

    assertThat(user).isNotNull();
    assertThat(user.getId()).isNotNull();
    assertThat(user.getName()).isEqualTo("Tom");
}
```

比：

```java
assert user != null;
```

更专业。

## 方案3：Mock测试（企业常用）

不连数据库。

测试：

```java
UserServiceImpl
```

业务逻辑。

例如：

```java
@ExtendWith(MockitoExtension.class)
class UserServiceImplTest {

    @Mock
    private UserMapper userMapper;

    @InjectMocks
    private UserServiceImpl userService;

    @Test
    void getUserById() {

        User user = new User();
        user.setId(1L);
        user.setName("Tom");
        user.setAge(18);

        when(userMapper.selectById(1L))
                .thenReturn(user);

        UserVO result = userService.getUserById(1L);

        assertThat(result.getName())
                .isEqualTo("Tom");
    }
}
```

这种：

```text
不启动Spring
不连接数据库
执行速度快
```

企业里数量最多。

## 你的代码还有个问题

你继承了：

```java
extends ServiceImpl<UserMapper, User>
```

实际上已经自带：

```java
save()
getById()
list()
removeById()
updateById()
```

所以：

```java
@Autowired
private UserMapper userMapper;
```

很多时候都不用写。

可以直接：

```java
@Override
public UserVO getUserById(Long id) {

    User user = this.getById(id);

    return user == null
            ? null
            : convertToVO(user);
}
```

或者：

```java
@Override
public List<UserVO> getAllUsers() {

    return this.list()
            .stream()
            .map(this::convertToVO)
            .toList();
}
```

因为：

```java
ServiceImpl
```

里面已经帮你注入了：

```java
protected M baseMapper;
```

等价于：

```java
baseMapper.selectById()
baseMapper.selectList()
```

所以你现在这个类更像是：

```java
MyBatis-Plus + 原生MyBatis混着写
```

学习阶段没问题，但如果要发挥 `IService + ServiceImpl` 的价值，后面可以逐渐改成用：

```java
this.getById()
this.save()
this.list()
```

而不是自己再注入 `UserMapper`。这样代码会少很多。
