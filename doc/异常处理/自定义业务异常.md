`BusinessException` 是**自定义业务异常**，用于在业务逻辑中主动抛出错误。

## 作用

| 场景       | 示例                                           |
| ---------- | ---------------------------------------------- |
| 用户不存在 | `throw new BusinessException("用户不存在")`    |
| 余额不足   | `throw new BusinessException(400, "余额不足")` |
| 权限不够   | `throw new BusinessException(403, "无权访问")` |
| 重复操作   | `throw new BusinessException("用户名已存在")`  |

## 核心代码

```java
public class BusinessException extends RuntimeException {
    private final int code;

    public BusinessException(String message) {
        super(message);
        this.code = 500;
    }

    public BusinessException(int code, String message) {
        super(message);
        this.code = code;
    }

    public int getCode() {
        return code;
    }
}
```

## 使用示例

### 1. 在 Service 中抛出异常

```java
@Service
public class UserServiceImpl extends ServiceImpl<UserMapper, User> implements UserService {

    @Override
    public UserVO getUserById(Long id) {
        User user = userMapper.selectById(id);
        if (user == null) {
            throw new BusinessException("用户不存在");
        }
        return convertToVO(user);
    }
}
```

### 2. GlobalExceptionHandler 自动捕获

```java
@ExceptionHandler(BusinessException.class)
public ApiResponse<?> handleBusinessException(BusinessException e) {
    log.warn("业务异常: code={}, message={}", e.getCode(), e.getMessage());
    return ApiResponse.error(e.getCode(), e.getMessage());
}
```

### 3. 前端收到统一格式响应

```json
{
  "code": 404,
  "message": "用户不存在",
  "data": null
}
```

## 优势

| 优势           | 说明                                         |
| -------------- | -------------------------------------------- |
| **统一管理**   | 所有业务异常都用同一个类                     |
| **携带状态码** | 可以自定义 HTTP 状态码                       |
| **日志记录**   | GlobalExceptionHandler 自动记录日志          |
| **解耦**       | Service 层只关心业务逻辑，不关心如何返回错误 |

## 和普通 Exception 的区别

```java
// ❌ 不推荐：直接抛出 RuntimeException
throw new RuntimeException("用户不存在");

// ✅ 推荐：使用 BusinessException
throw new BusinessException("用户不存在");
```

| 特性       | RuntimeException | BusinessException |
| ---------- | ---------------- | ----------------- |
| 语义       | 通用异常         | 业务异常          |
| 状态码     | 固定 500         | 可自定义          |
| 日志级别   | ERROR            | WARN              |
| 是否预期内 | 否               | 是                |

**总结**：`BusinessException` 让你在代码中**主动、明确**地抛出业务错误，而不是被动地捕获系统异常。
